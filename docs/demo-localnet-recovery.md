# Local demo recovery policy

Local LEZ demos use fake funds and a disposable chain. Prefer a known-good fixture over
debugging stale state. Public testnet operators should not follow this reset-first policy;
see Step 12 runbook testnet notes in [`step12-user-eligibility.md`](step12-user-eligibility.md).

Related: [`step10a-handoff-and-follow-up.md`](step10a-handoff-and-follow-up.md),
[`step10a-local-chain-fixture.md`](step10a-local-chain-fixture.md).

## Principle

- Treat on-chain localnet state and `fixtures/localnet.json` as disposable.
- Treat `payment_streams_state.json` (logoscore `instancePersistencePath`) as disposable
  for every demo run unless you intentionally continue one session.
- Keep wallet keys and installed `.lgx` modules across resets when they still work.
- When unsure, or when verification fails, reset the chain and re-seed instead of tuning
  demo-only env bypasses.

`PAYMENT_STREAMS_ALLOW_DEPLETED_STREAM_PROOF` is an escape hatch when chain writes are broken
but you still need to show proof bytes. It is not the normal local demo path.

## Demo modes

| Mode | Goal | Chain | Module off-chain state |
| --- | --- | --- | --- |
| Smoke | Mapping, reads, `stream_proof` on seeded stream `0` | Reuse if Step 10a verify passes and stream `0` has unaccrued balance | New `--persistence-path` each run |
| Full arc | Proposal, `createStream`, proof; Step 13 prepare ŌåÆ verify | Fresh seed recommended | Always wipe before the arc |

Stream `0` on a long-lived localnet accrues until `STREAM_DEPLETED`. That is correct policy,
not a product bug. Recovery is reset and re-seed, not depleted bypass by default.

## What to keep vs wipe

Keep across local resets (slow or stable):

- Patched `logos_execution_zone` and `payment_streams_module` under `MODULES`.
- `PAYMENT_STREAMS_GUEST_BIN` pointing at the built guest `.bin`.
- `.scaffold/wallet/wallet_config.json` and valid `storage.json` when deploy and `open` work.

Wipe on blank-slate recovery:

| Path | Reason |
| --- | --- |
| `.scaffold/state/` | Chain age, failed txs, program id drift |
| `fixtures/localnet.json` | Regenerated by seed; must match deployed program |
| `.scaffold/step12-persist*` | Stale peer mapping, session keys, inventory |
| `.scaffold/step13-persist*` | Stale provider acceptance rows for verify demos |
| `.lez_payment_streams-state` | Owner signer drift after wallet reinit |

Optional wipe on full blank slate:

| Path | Reason |
| --- | --- |
| `.lez_payment_streams-fixture-provider` | New provider line in manifest |
| `.scaffold/wallet/storage.json` | Invalid format (`missing field sk`, etc.) ŌĆö use reinit |

## Recovery ladder

1. Soft ŌĆö New logoscore run with a fresh `--persistence-path`, re-register provider mapping,
   check stream status via `listMyStreams` or reads.
2. Re-seed ŌĆö Same chain: `make build deploy`, `./scripts/seed-localnet-fixture.sh`, clear
   module persist dirs, refresh manifest.
3. Blank slate ŌĆö `./scripts/demo-localnet-fresh.sh` (stop localnet, wipe chain state, clear
   Step 12 persist, seed, run Step 10a verify). Use `REINIT_WALLET=1` when wallet storage
   or deploy fails.
4. Run demo ŌĆö `logoscore` with a new `--persistence-path`, `open` wallet, then Step 12 calls.

Signals to jump to step 3:

- `./scripts/verify-step10a-dod.sh` fails.
- `lgs localnet start` fails with missing `sequencer/service/configs/debug/sequencer_config.json`
  after Step 11d pin bump ŌĆö run `./scripts/ensure-scaffold-lez-layout.sh` (links `sequencer` ŌåÆ `lez/sequencer` in the LEZ cache).
- `prepareEligibilityForStoreQuery` returns `STREAM_DEPLETED` on manifest stream `0` and you
  need an honest `stream_proof` without bypass.
- `make deploy` or `storage.json` parse errors.
- Manifest program id stale after `make build`.

## One-shot blank slate

From repo root:

```bash
chmod +x scripts/demo-localnet-fresh.sh
./scripts/demo-localnet-fresh.sh
```

Wallet storage broken:

```bash
REINIT_WALLET=1 ./scripts/demo-localnet-fresh.sh
```

Skip Step 10a verify at the end (faster, after you trust seed):

```bash
SKIP_VERIFY=1 ./scripts/demo-localnet-fresh.sh
```

Then Step 12 smoke (set `PAYMENT_STREAMS_GUEST_BIN` to the built guest `.bin`):

```bash
export PAYMENT_STREAMS_GUEST_BIN="$PWD/methods/guest/target/riscv32im-risc0-zkvm-elf/docker/lez_payment_streams.bin"
export PERSIST_DIR="$PWD/.scaffold/step12-persist-$(date +%s)"
./scripts/verify-step12-dod.sh
```

After a fresh reset, require honest `stream_proof` in DoD smoke (fail on depleted instead of skip):

```bash
REQUIRE_STREAM_PROOF=1 ./scripts/verify-step12-dod.sh
```

Or offline only:

```bash
VERIFY_LOGOSCORE=0 ./scripts/verify-step12-dod.sh
```

Step 13 provider cross-test (after fresh reset when stream `0` is not depleted):

```bash
export PERSIST_DIR="$PWD/.scaffold/step13-persist-$(date +%s)"
VERIFY_LOGOSCORE=1 ./scripts/verify-step13-dod.sh
```

See [`step13-provider-eligibility.md`](step13-provider-eligibility.md).

## Scripts

| Script | Role |
| --- | --- |
| `scripts/demo-localnet-fresh.sh` | Blank-slate chain + seed + optional wallet reinit + 10a verify |
| `scripts/clear-demo-module-persist.sh` | Remove known Step 12 logoscore persist dirs under `.scaffold/` |
| `scripts/reinit-scaffold-wallet.sh` | Recreate scaffold wallet storage (does not reset sequencer state alone) |
| `scripts/ensure-scaffold-lez-layout.sh` | Symlink `sequencer` for LEZ 510+ layout after `lgs setup` |
| `scripts/seed-localnet-fixture.sh` | Idempotent on-chain seed when chain is already up |

## Testnet contrast

On public testnet, persist manifest, vault, and operator wallet. Reset is exceptional.
Local policy does not apply there.
